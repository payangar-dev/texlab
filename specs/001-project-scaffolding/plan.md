# Implementation Plan: Project Scaffolding

**Branch**: `001-project-scaffolding` | **Date**: 2026-03-28 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-project-scaffolding/spec.md`

## Summary

Initialize a Tauri v2 desktop application shell with React 19 + TypeScript 5 + Vite 6 frontend and Rust backend. The backend follows strict clean architecture with 5 distinct layers (domain, use_cases, infrastructure, commands, mcp). The frontend uses a standard component-based structure. Both `CLAUDE.md` and `CONTRIBUTING.md` document conventions. The app launches a blank window titled "TexLab" with no content — the minimum viable shell for subsequent feature development.

## Technical Context

**Language/Version**: Rust ≥ 1.77 (backend), TypeScript ^5.7 (frontend), Node.js ≥ 20 LTS
**Primary Dependencies**: tauri ^2.10, react ^19.2, vite ^6.0, zustand ^5.0 (see constitution for full list)
**Storage**: File system only (project files, .texlab archives — future features)
**Testing**: `cargo test` (Rust), vitest + jsdom + @testing-library/react (frontend)
**Target Platform**: Windows 11 (primary), cross-platform compatible (no platform-specific workarounds)
**Project Type**: Desktop application (Tauri v2)
**Performance Goals**: Window appears within 10 seconds of `npm run tauri dev` (SC-002)
**Constraints**: Zero warnings from `cargo build` + `npm run build` (SC-005)
**Scale/Scope**: Single-user local application, ~15 source files for scaffolding
