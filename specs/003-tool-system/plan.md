# Implementation Plan: Tool System — Brush, Eraser, Fill, ColorPicker, Line, Selection

**Branch**: `003-tool-system` | **Date**: 2026-03-28 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/003-tool-system/spec.md`

## Summary

Implement six drawing tools (Brush, Eraser, Fill, Color Picker, Line, Selection) as pure domain logic with a unified press-drag-release interaction model. Each tool operates on a `PixelBuffer` through a shared `ToolContext`, using Bresenham's algorithm for line drawing and stroke interpolation, and BFS flood fill for the fill tool. The system adds a `Tool` trait, value objects (`BrushSize`, `Selection`), and a minimal modification to `Layer` (`buffer_mut()`).

## Technical Context

**Language/Version**: Rust ≥ 1.77
**Primary Dependencies**: None (domain layer — `std` only, including `std::collections::VecDeque` for BFS)
**Storage**: N/A (in-memory pixel operations)

**Target Platform**: Cross-platform (desktop via Tauri, but this layer is platform-independent)
**Project Type**: Desktop application — domain layer module
**Performance Goals**: Synchronous tool application. Fill on 64×64 must complete instantly (SC-003).
**Constraints**: Zero external dependencies in domain. All tools testable without I/O.
**Scale/Scope**: 6 tools, ~8 new files, ~500-700 lines of production code + comprehensive tests
